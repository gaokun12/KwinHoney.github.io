3 numbers – LeetCode #16

Some quizs about three numbers in LeetCode.

The question:

Given an integer array nums, return all the triplets [nums[i], nums[j], nums[k]]such that i != j, i != k, and j != k, and nums[i] + nums[j] + nums[k] == 0.

Notice that the solution set must not contain duplicate triplets.

The method:
  • The three numbers problem needs a iterator to travel all numbers in the list (call a main interator). Besides, we also need two other iterators to perform the add operation. The number of all iterators are three. Hence, the time complexity is $O(n^2)$.
  • How to find the right solution?
    • Do the sort for the given list ($O(n \cdot \text{log}n)$);
    • We compare the addition between target and the current additon. If the current addition less than the target, then we move the middle iterator to move to the right side. Similiarily, if the current addition large thant the target, then we move the rightest iterator move to the left side.
    • We need a distanace value to measure the distance betweent he current addition with the target.

The problem:

Given an array nums of n integers, return an array of all the *unique** quadruplets*[nums[a], nums[b], nums[c], nums[d]] such that:

  • 0 <= a, b, c, d < n
  • a, b, c, and d are distinct.
  • nums[a] + nums[b] + nums[c] + nums[d] == target

You may return the answer in any order.

The idea:

We just use four iterators to search the whole list. However, the time complexity is not $O(n^2)$ in this situation. It turns $O(n^3)$ because we use two main iteraors during the search process.
